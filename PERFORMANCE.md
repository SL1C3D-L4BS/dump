## 4. Architecture notes

- **X12Reader:** Tree and segment index are reset on each `ST`; one row is yielded per `CLM` via `copyTree`. No retention of prior transactions → O(1) heap.
- **MapStream:** Line-by-line scan; no accumulation of rows in memory.
- If HeapAlloc were ever observed to grow linearly, the fix is to ensure the internal tree is fully cleared and GC-ready after every Yield (e.g. explicit nil of tree reference after copy and before next segment).
